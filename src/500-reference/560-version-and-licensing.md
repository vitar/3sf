It ensures transparency, traceability, and consistency across all derivative works, training materials, and digital publications.

### Version Summary

Version 1 consolidates all theoretical and practical layers of the 3SF Framework, including:

- 3-in-3 Model (Client ↔ Vendor ↔ Product System)
- 3SF Theory Layers (Stable Rules Layer, Contextual Drivers Layer, and Contracts Architecture)
- 3SF Practice Artifacts (Contracts, Canvases, Metrics, and Indexes)
- 3SF Reference Materials (Acronyms, Glossary, Sources, and Frameworks)

This version establishes the **core terminology, structure, and systemic relationships** that will remain stable across future revisions.

## Versioning Policy

- **Major versions (V2, V3, …)** introduce structural or conceptual extensions to the model.
- **Minor revisions (e.g., V1.1)** include clarifications, formatting, or terminology alignment updates.
- All published versions will remain **permanently available** for reference and citation.
- Future releases will maintain backward compatibility for all key 3SF concepts and relationships.

## Attribution Guidelines

If reusing or adapting 3SF content:

1. Include a visible credit line referencing *3in3.dev* and the license type.
2. Retain section numbering where possible to preserve internal navigation and cross-references.
3. When combining 3SF content with other frameworks, clearly separate attribution and derived materials.
4. For translations or derivative works, add a note identifying the adaptation (e.g., “Translated from the original 3in3.dev V1 documentation licensed under CC BY 4.0”).
